## Jump Logbook

Install dependencies and start the local development server:

```sh
pn install
pn dev
```

## End-to-end tests

Run the Playwright end-to-end test suite:

```sh
pn test:e2e
```

Tests start the application on `127.0.0.1:8788` and reset the isolated local D1
database in `.playwright/state/` before each run.

## Local database (Cloudflare / Wrangler)

The local D1 database is stored by Wrangler in `.wrangler/state/v3/d1/`.
The SQLite filename is generated by Wrangler, so use this directory rather than
depending on a specific filename. The entire `.wrangler/` directory is ignored
by Git and can be removed to reset local database state.

Generate a Drizzle migration after changing the schema:

```sh
pn db:generate
```

Initialize or update the local D1 database with Drizzle migrations:

```sh
pn db:migrate
```

Insert or update the development user. This command initializes the database
when needed.

```sh
pn db:dev-user
```

The development user credentials are `developer` and `developer123`.

List users in the local database. This command initializes the database when
needed.

```sh
pn db:list-users
```

Create or update an invitation code. Prompts for local or remote database,
code, and number of uses.

```sh
pn db:create-invitation
```

## Cloudflare

Generate or update TypeScript types for Cloudflare bindings:

```sh
pn cf-typegen
```

Deploy the application:

```sh
pn deploy
```

## Self-host without Cloudflare (Node + SQLite)

You can run the same app on Node with a local SQLite file. Cloudflare Workers +
D1 remain fully supported; this is an alternate runtime entrypoint.

### Requirements

- Node.js (see `mise.toml`)
- Native build tools for `better-sqlite3` (usually prebuilt binaries)

### Build and run

```sh
pn install
pn build:node
pn db:dev-user:sqlite   # optional: seed developer / developer123
pn start
```

Then open http://127.0.0.1:8787

`pn start` runs migrations on the SQLite file at startup.

### Environment

| Variable      | Default             | Description                                          |
| ------------- | ------------------- | ---------------------------------------------------- |
| `PORT`        | `8787`              | HTTP port                                            |
| `HOST`        | `127.0.0.1`         | Bind address (`0.0.0.0` to listen on all interfaces) |
| `SQLITE_PATH` | `data/hypyt.sqlite` | Path to the SQLite database file                     |

Example:

```sh
HOST=0.0.0.0 PORT=3000 SQLITE_PATH=/var/lib/hypyt/db.sqlite pn start
```

### SQLite database helpers

```sh
pn db:migrate:sqlite      # apply Drizzle migrations only
pn db:dev-user:sqlite     # migrate + upsert developer user
```

The SQLite file and `data/` directory are gitignored. Delete `data/` to reset
self-hosted state.

Session cookies use `Secure` only on HTTPS. Plain HTTP self-host works on
localhost without extra configuration.

### Production notes

- Keep `drizzle/` next to the process (migrations run from that folder).
- Serve behind a reverse proxy (Caddy, nginx, etc.) for TLS in production.
- `pn deploy` continues to target Cloudflare Workers + D1 as before.
